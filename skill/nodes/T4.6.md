# T4.6 - Dev 部署 + QA 测试

前后端开发完成、T4.5 契约校验通过后，在本地 Dev 环境部署，QA 开始人工接口测试。

**此阶段与 T6 自动化门禁并行执行 — QA 手工测试和自动化脚本同时跑，互不等待。**

## ✅ 完成标准（DoD）

| 检查项 | 说明 |
|--------|------|
| Dev 环境部署完成 | 后端 + 前端服务均 running |
| QA 人工接口测试通过 | 核心接口响应正确 |
| Bug 已记录 Kanban | P0/P1 Bug 有对应任务 |

## 📦 部署检查

```bash
# 后端
cd backend
cargo build --release
./target/release/quant-trading    # 或 docker-compose up

# 前端
cd frontend
npm run dev

# 确认服务健康
curl http://localhost:8080/health     # 后端
curl http://localhost:5173            # 前端
```

## 🧪 QA 人工测试清单

### 核心接口测试（必须全部覆盖）

| 优先级 | 测试项 | 方法 |
|--------|--------|------|
| P0 | 所有 API 端点可调通 | Postman / curl |
| P0 | 请求参数校验生效 | 故意传错参数验证错误返回 |
| P0 | 响应 JSON 格式正确 | 对比 TypeScript 类型 |
| P1 | 前后端数据一致性 | 页面显示 = 数据库实际值 |
| P1 | 错误码前后端一致 | 故意触发错误，对比前端提示 |
| P1 | 数据库读写正确 | 增删改查实际验证 |
| P2 | 边界条件 | 空数组、超长字符串、并发请求 |

### 冒烟测试

```bash
# 快速验证核心链路
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"wrong"}' \
  # 期望: 401 Unauthorized

curl http://localhost:8080/api/v1/health
  # 期望: 200 OK
```
