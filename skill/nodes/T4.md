# T4 - 并行开发

前后端根据 T2 的 API 契约并行开发，互不阻塞。

## ✅ 完成标准（DoD）
- 后端所有 API 接口实现完成
- 前端所有页面和组件开发完成
- **后端：每个功能模块编写 `#[cfg(test)]` 单元测试，覆盖率 ≥ 70%**
- **前端：每个组件编写 Vitest/Jest 单元测试，核心组件覆盖率 ≥ 60%**
- 单元测试必须随功能代码一起提交（不允许先合代码后补测试）
- ESLint / Clippy 无错误
- 代码已提交 PR

## 📝 开发规范（强制要求）

### 后端 — Rust 测试驱动开发

```bash
# 1. 先写测试，再写实现（TDD 建议）
backend/src/handlers/user.rs

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_create_user_success() {
        // 测试代码
    }
}

# 2. 实现功能代码后，运行测试
cargo test -p <package>

# 3. 检查覆盖率
cargo llvm-cov --html
# 打开 target/llvm-cov/html/index.html 查看行覆盖率
```

### 前端 — Vitest 组件测试

```bash
# 1. 组件目录结构（测试文件与组件同目录）
frontend/src/components/
├── UserCard.vue
└── UserCard.test.ts   # ← 必须一起提交

# 2. 测试文件规范
import { describe, it, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import UserCard from './UserCard.vue'

describe('UserCard', () => {
    it('renders user name correctly', () => {
        // ...
    })
})

# 3. 运行测试
npx vitest run src/components/
```
