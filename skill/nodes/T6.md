# T6 - QA 测试

自动化门禁检查 + 安全扫描，交付前的最后一道防线。

## ✅ 完成标准（DoD）

| 检查项 | 判定 |
|--------|------|
| 安全扫描 | Critical = 0，High ≤ 3 |
| 后端测试 | `cargo test` 全通过 |
| 前端测试 | `vitest` / `jest` 全通过 |
| API 冒烟测试 | `/health` 等端点 HTTP 200 |

## 🤖 自动化检查（4 项）

```
./scripts/t6-qa-check.sh [project_root]
```

| 步骤 | 检查内容 | 失败处理 |
|------|---------|---------|
| 1/4 安全扫描 | `security-scan.sh` 自动执行 5 维度检查 | Critical → FAIL |
| 2/4 后端测试 | `cargo test --quiet` | 失败 → FAIL |
| 3/4 前端测试 | `vitest run` / `jest` | 失败 → FAIL |
| 4/4 API 冒烟测试 | `curl /health` 等端点 | 全部无响应 → WARN |

## 📋 安全扫描 5 维度（自动执行）

| 维度 | 工具 |
|------|------|
| Rust 依赖漏洞 | `cargo audit` |
| NPM 依赖漏洞 | `npm audit --json` |
| 硬编码密钥 | `grep` 模式扫描 |
| SQL 注入风险 | 字符串拼接 SQL 检测 |
| CORS 配置 | `AllowAnyOrigin` 检测 |

## ⚠️ 注意：P0/P1 Bug 仍需人工录入

自动化脚本只检查**代码质量门禁**（测试通过、安全扫描）。

**P0/P1 Bug 数量统计** 由 QA Lead 手工录入 Kanban，
脚本不替代人工 QA 测试和 Bug 录入。

## 🔙 回滚触发条件

- Critical 安全漏洞存在
- `cargo test` / `vitest` 有测试失败

回退到 **T5** 修复。
